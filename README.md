# CrackTiers

**India's Premier Government Exam Question Bank** — practice 10,000+ bilingual MCQs for SSC, UPSC, Railway, Banking, Defence, Teaching, and State PSC exams.

Built with **Next.js 16**, **React 19**, and **Tailwind CSS 4**.

---

## Features

- **Question Bank** — 10,200+ MCQs in English and Hindi across Reasoning, Maths, English, and GK / Current Affairs (2025–2026)
- **Online Test Series** — timed mock tests for SSC, UPSC, Railway, Banking, Defence, Teaching, Rajasthan, and UP
- **Instant feedback** — tap an option to check answers with explanations
- **Bilingual UI** — switch between English and Hindi anytime
- **Smart filters** — search, subject, category, difficulty, and year filters with mobile-friendly pills
- **Free tier** — 30 free questions and limited mock tests for guests; full access after free registration
- **User dashboard** — track progress and manage your profile
- **SEO ready** — sitemap, robots.txt, JSON-LD, and subject/category landing pages

## Getting Started

### Prerequisites

- **Node.js** 18.18+ (20+ recommended)
- **npm**, **yarn**, or **pnpm**

### Installation

```bash
git clone <your-repo-url>
cd CrackTiers
npm install
```

### Generate the question bank

Questions are generated before each build. Run manually for local development:

```bash
npm run generate:questions
```

This writes `data/generated/questions.json` (~10,200 questions).

### Development

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### Production

```bash
npm run build
npm start
```

## Environment Variables

Create a `.env.local` file in the project root (optional):

```env
NEXT_PUBLIC_SITE_URL=https://cracktiers.in
```

Used for SEO metadata, sitemap, and canonical URLs. Defaults to `https://cracktiers.in` if unset.

## Question Data

Questions are generated by `scripts/generate-questions.mjs` and stored in compact JSON format. Each question includes:

- English and Hindi question text
- Four options (A–D) in both languages
- Correct answer, explanation, subject, category, difficulty, and year

To regenerate after changing the script:

```bash
npm run generate:questions
```

Restart the dev server if it is already running.

---

## Auth (Demo)

Authentication is client-side for demo purposes:

- **Register** — creates a user in `localStorage`
- **Login** — any email/password works; existing profile is preserved by email
- **Profile** — edit name, email, phone, location, and target exam at `/profile`

Replace `lib/auth-context.tsx` with a real backend (NextAuth, Clerk, Firebase, etc.) for production.

## Deployment

Deploy to [Vercel](https://vercel.com) or any Node.js host:

1. Set `NEXT_PUBLIC_SITE_URL` to your production domain
2. Run `npm run build` (question generation runs automatically via `prebuild`)
3. Start with `npm start` or use the platform's Next.js preset
